今天，我终于冲出了外包的怪圈

今天，我终于冲出了外包的怪圈

0. 序

今年八月，由于疫情以及自身原因，每个工作都很难找，我大致上投了 50 多个简历，面试了不超过10家公司，终于有一个在我的期望内：非外包、人数多、比较知名。虽然工资比我预期的少了一些，但是其他地方已经非常满意了。

当然，由于我自己的缘故，BAT 是异想天开的事情，网易阅文都不鸟我。

说外包是个天坑，真的一点都没错，每次面试的时候，在介绍自己的项目时总会看到面试官一脸嫌弃的目光，仿佛在说：“就这？”。年少轻狂时，一直认为外包只是人员派遣罢了，项目外包并不是外包。但是现在看了，现实给了我狠狠的一棒，只有公司有自己的产品，才不算是外包。

更令人羞愧的是，看到简历上的增删改查，我都一时哽咽，所以我特别羡慕那些刚毕业的孩子们，他们的难度比我要小很多。前几天去北京面试的时候，碰到了一个刚毕业的，不禁感叹：“校招，好好珍惜吧！”

1. 注意事项 & 准备

准备是面试前最最重要的，如果是走社招，千万要记得，不要像我一样很勇，搞裸辞这种东西，实在是太傻了。可以先收藏自己心仪的公司，先投一些不是特别大型的公司，像我一开始就投错了，现在没机会了(￣、￣)

2020年，两年开发经验需要掌握什么？时代千变万化，几年之后的两年工作经验，估计需要比我掌握的还要多，还要复杂，所以这里加了一个 2020 年的前缀，以防误导看到我这篇博客的人。虽然可能过几年之后，就不是这里所描述的了，但是你还是可以通过一些企业（非驻场企业要求通常符合行情，但另一个情况是像华为虽然要求写的少，但去了之后还是老 N 样。所以，可以看一些中小型企业的岗位要求）的招聘信息了解一二。如下所示：

刚入行的小伙伴们，可能就会造成滴滴的要求比较低的假象，其实并不是如此，滴滴、华为、淘宝的要求绝对没有这么简单，但是，希望大家还是能甘于去投大公司的简历。但是简历投放先后顺序我建议是先小后大。（像我之前不是很敢投这些大公司，觉得投了也面不上，但是机会绝对是自己去争取的）。

刚才有点说偏了，继续聊两年岗位需求的事情，通过第一张照片，不难看出，目前两年的开发经验需要掌握：

  • 熟悉 Java（这里面包括 Java 基础、反射、代理、泛型、集合、IO、多线程）
  • 熟悉常用的 Java 框架（例如 Spring、Spring Boot、MyBatis）
  • 了解 JVM（如数据区域、类加载机制、垃圾回收算法、JMM，JVM 优化）
  • 良好的面向对象设计理解（其实就是在说需要掌握 23 中设计模式）
  • 中间件、分布式技术（如 Redis、MQ、Spring Cloud、Zookeeper、ES、Docker、雪崩、降级、熔断）
  • 常用数据库开发经验（例如 MySQL、Oracle）
  • 了解 maven、GIT 命令

还有很多公司还会需要掌握：

  • 数据结构和算法（例如 AVL、红黑、B+树、队列、bitmap、力扣简单程度的算法）
  • HTTP 基本协议以及 TCP/IP

讲真，在现在看来，这里面的很多东西，我还没有掌握……两年之前我对这些只是嗤之以鼻，觉得这些都是造火箭的东西，不去也罢，没想到……每个公司都会问，真的。

2. 有迹可循

因为我不了解中间件、分布式这些东西，所以面试的时候，面试官都会问我其他内容了，比方说，Java 基础、框架、JVM、设计模式这些，下面给大家说说一个月以来大家都在问什么：

2.1 基础

  • Java 特性，Java 基本类型以及占用的字节数
  • Object 常用方法：equals、hashcode、wait、notify、notifyAll、toString、finalize、getClass
  • 为什么重写完 equals 还要重写 hashcode？
  • static 几种用法（类似于孔乙己中问茴香豆有几种写法）
  • final
  • String 常用方法，有个面试官问我 "a,".split(",")，",a".split(",")区别
  • 重写和重载的区别
  • 受检异常和非受检异常
  • 反射机制，常用反射类
  • 泛型上下限通配符，桥方法，泛型需要注意的点
  • 几个内部类区别：普通、匿名、静态、局部
  • 动态代理，经常会问。还会问 Java 动态代理和 CGLIB 区别
  • 集合：ArrayList 初始化过程，扩容机制；HashMap 初始化过程，初始容量，扩容机制、为什么扩容两倍；ConcurrentHashMap 为何是线程安全的？HashSet 底层是什么？（不按常理出牌的面试官）LinkedList 、ArratyList、Vector 相同点以及不同点
  • 还是集合：HashMap 是否允许空值？HashMap 与 HashTable 区别？
  • IO：问的不多，有一个笔试题写一个文件流转字节，但是估计大厂会问 NIO 和 IO 的区别
  • 数据库：经常会在笔试题让你写原生 Java 查询数据库，考查基础知识点

2.2 多线程

  • 说一说什么是多线程（致命问题）

  • 线程的几大状态，创建线程的几种方式

  • 进程和线程有什么区别，并发和并行的区别，并发和多线程的区别

  • 多线程用到哪些类（底层关键字 volatile、synchronized，还有 CAS 思想； AQS、Lock、BlockingQueue、并发集合、原子类）

  • volatile、synchronized 底层（没听错，经常会问底层，虽然我不知道有什么用）

  • synchronized 锁方法和锁代码块的区别

  • 什么是 CAS？ABA 具体是在说什么

  • AQS 底层原理

  • Lock 和 Synchronized 区别

  • Lock 几种实现以及原理

  • 原子类几种实现以及原理

  • 并发集合（ConcurrentHashMap，CopyOnWriteList，ConcurrentLinkedQueue，BlockingQueue等）

  • 几种线程池，以及底层实现（核心线程数、最大线程数、线程拒绝策略、阻塞队列这些都是重点）

  • 信号量、计时器、栅栏

  • ThreadLocal 理解、实现方式、使用场景

  • 与多线程密切相关的单例模式

2.3 设计模式

如果你在简历上写设计模式（即使你不写），面试官也会问你设计模式，还好之前的公司员工有让我了解过，这里需要感谢他一下。常见的会问到：

  • 单例模式（懒汉、饿汉、枚举、内部类），为什么懒汉要用双锁，什么情况下会用到单例模式
  • 策略模式
  • 建造者模式（因为我简历里写了，所以就问到了）
  • 装饰者模式
  • 适配器模式
  • 观察者模式
  • 工厂和抽象工厂模式
  • 模板模式
  • 代理模式（静态代理、动态代理区别也会问）

希望大家不要死记硬背设计模式，要不然非常容易露馅。知其然知其所以然才是最重要的。

2.4 Spring 和 Spring Boot

Spring 面试题会碰到很多，不按套路的面试官会问 Spring Boot 相关的内容。

  • Spring 作用，两大特性
  • Spring 常用注解
  • 自己的语言介绍一下 IOC
  • AOP 术语：切面、切点、横切点、前置、后置、环绕
  • AOP 实现原理，CGLIB 与 Java 动态代理区别
  • Spring Bean scope
  • Spring Bean 生命周期（这个真的很恶心）
  • Spring 与 Spring Boot 区别是什么
  • Spring Boot 常用注解（注意不是 Spring 注解）
  • @Controller 、@Service、@Repository相同点和不同点
  • @Controller、@RestController 区别
  • 事务传播机制以及 @Transactional 常用属性

2.5 数据库

由于面试上写的 MySQL，所以面试官是以这个为主问的。

  • 聚簇索引和非聚簇索引区别
  • 索引的几种类型
  • InnoDB 和 MyISAM 区别
  • 乐观锁、悲观锁、共享锁、排他锁、表锁、行锁
  • 什么情况下用行锁，什么情况下用表锁
  • 隔离级别以及脏读、幻读、不可重复读
  • 为什么需要索引，如何理解索引优缺点
  • 索引失效场景
  • Explain
  • LEFT JOIN、LEFT OUTER JOIN、RIGHT JOIN、INNER JOIN 区别
  • 数据库三范式
  • 如何设计数据库（多对多、一对多如何设计；以及一些数据类型的优化）
  • B+ 树，为什么用 B+ 树（这里又引出了数据结构）

2.6 数据结构及算法

数据结构在中小厂面试中遇到的几率比较少，但是笔试题经常会放一些算法题。

  • 常见的数据结构（重点看下 bitmap）
  • 树、二叉树、平衡树、红黑树（为什么用红黑而不用平衡树；不过还好没让写红黑树变色以及左右旋转）
  • B 树和 B+ 树区别
  • 常见排序以及时间复杂度
  • 算法就没有具体的问了，刷下力扣就可以了，中小厂一般都是简单程度的，当然大家不要觉得简单就不去刷

2.7 MyBatis

由于我简历写的是 Hibernate 和 Spring data JPA，所以这里就不写 MyBatis 的面试题目了。

2.8 JVM

虽然我简历没有写 JVM，但是面试官还是会问。由于我直接说不了解，所以这里就放一些在看准上看到的一些 JVM 面试题目：

  • 垃圾回收算法
  • JVM 内存模型
  • JVM 中对象存储空间
  • 什么情况下会栈溢出，什么情况下会 OOM
  • JVM 优化
  • JVM 偏向锁、轻量级锁、重量级锁

2.9 逻辑思维题（日了狗）

  1. 两个蜡烛，一根蜡烛燃烧完需要一个小时，问如何让两根蜡烛在四十五分钟燃烧完
  2. 六个水杯，有水有水有水无水无水无水，移动一个水杯，变成：有水无水有水无水有水无水
  3. 三门问题
  4. 20 张光滑的扑克牌，八张正面朝上，其余朝下。怎么样在闭着眼睛的情况下将扑克牌分成两组，使两组正面朝上的数量一样多？

3. 面试总结

面试造火箭，入职拧螺丝。在疫情当下，工作可谓是相当的难找。不难看出，很多题目都有一种“学术”在里面，例如什么底层，什么垃圾回收机制，什么 B+ 树。我们不仅要去应对这些内容，我们还要理解它们。为什么不建议大家去外包公司？因为没有特别新的技术，你看我不了解中间件，不了解消息队列，不了解微服务，真的很难去应聘合适的工作。我之前的公司没有这种场景去使用这些，所以根本没办法接触这些东西。像多线程这些都是我自学的。当面试官问到使用多线程场景的时候，我基本都说不上来。

环境确实非常重要，我之前一直认为，JVM 这种东西最起码得高级才懂吧？多线程底层最起码三年以上才知道吧？原来，一两年的工作经验也需要掌握这些内容。

4. 给新人推荐几本书

买技术书买新不买旧，买国外不买国内，书的评价好不好最好使用豆瓣阅读查一下。这几本书都是我翻过觉得不错的：

  • 《Head First Java》

  • 《Java 核心技术》

  • 《Java8 实战》

  • 《算法》

  • 《Effective Java》

  • 《设计模式之禅》

  • 《Java 高并发程序设计》

  • 《深入理解 Java 虚拟机》

  • 《高性能 MySQL》
